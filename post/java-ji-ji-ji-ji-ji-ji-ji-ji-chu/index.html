封装

将一个对象私有化同时提供可被外界访问的属性和方法

继承

  • 子类拥有父类非 private 的属性和方法
  • 子类可以拥有自己的方法
  • 子类可以重写父类的方法

多态

引用变量所指向的具体类型和通过该引用变量发出的方法调用在编程时并不确定

跨平台

"一次编译, 到处运行"
编码时生成字节码文件(.class), 可在JVM上运行转换为原始代码

String

构造 实现

StringBuffer 方法调用了synchronized同步锁, 线程安全

不可变

  1. 保证 String 对象的安全性
  2. 保证 hash 属性值不会频繁变更, 确保了唯一性(HashMap实现key-value缓存)
  3. 可以实现字符串常量池

"+"拼接

编译器会自动转化为 StringBuilder , 但循环中会新建多个 StringBuilder

String.intern()

构造相同值得字符串对象返回相同的对象引用地址(数据过大会增加常量池负担)

静态方法不能调用非静态成员

非静态的实例成员变量属于对象, 而静态方法执行时并不一定存在对象

对象相等和指向他们的应用相等

对象相等 引用相等
比较内存中存放的内容是否相等 比较指向的内存地址是否相等

==和equals

  • ==
    • 基本数据类型比较值
    • 引用数据类型比较内存地址
  • equals
    • 比较的类没有重写equals方法, 则调用==比较
    • 类重写了equals, 比较两个对象的内容
    • String 中的 equals 没有被重写, 比较的是值

hashCode

获取对象的哈希码(散列码), 返回int, 确定该对象在哈希表中的索引位置

HashSet中的去重应用

add 时会先计算对象的 hashCode 来判断对象加入的位置,同时也会与其他已经加入的对象的 hashCode 作比较, 没有相符则假设对象没有重复出现. 如有相同则调用 euals 方法检查 hashcode 相等的对象是否真的相同, 减少 equals 次数, 提高执行速度

hashCode与equals的相关规定

  • 两个对象相等, hashCode一定相同
  • 两个对象相等, 对两个对象分别调用 equals 方法都返回 true
  • 两个对象有相同的 hashCode 不一定相等
  • equals 方法被重写则 hashCode 方法也必须重写
  • hashCode 的默认行为是对堆上的对象产生独特值, 如果没有重写 hashCode, 则该 class 的两个对象无论如何都不会相等

值传递

JAVA只有值传递(创建副本)

  • 一个方法不能修改一个基本数据类型的参数
  • 一个方法可以改变一个对象参数的状态
  • 一个方法不能让对象引用一个新的对象

final

final 关键字主要用在:变量, 方法, 类

  • 对一个 final 变量, 如果是基本数据类型的变量, 则其数值一旦在初始化之后就不能修改; 如果是引用类型的变量, 则在对其初始化之后便不能再让其指向另一个对象
  • 当用 final 修饰一个类时, 表明这个类不能被继承; final 类中的所有成员方法都会被隐式地指定为 final 方法
  • 锁定方法, 以防任何继承类修改它的含义

Integer 缓存

new Integer() 数值为 -128~127 时会缓存 Integer 对象

POJO类属性

必须使用包装类型

  • 基本类型有默认值, 0 和 null 还是有区别的
  • 数据库的查询结果有可能是 null, 因为自动拆箱, 用基本数据类型接收有NPE风险

GET&POST

GET POST
请求可被缓存 请求不可被缓存
请求保留在浏览器历史记录中 请求不会保留在浏览器历史记录中
请求可被收藏为书签 不能被收藏为书签
请求不应在处理敏感数据时使用 请求对数据长度没有要求
请求有长度限制(2048 字符) IE 和 Safari 限制2K Opera限制4K Firefox Chrome 限制8K
GET请求只应当取回数据
